1.
    在普通开发环境 -- 产出代码到内置服务器  fis3 release debug -c -wl

    -debug (代表不产出md5 ,不压缩)

    -c(代表清除缓存)，-w (代表监听) ，-l(代表浏览器自动刷新)


2.
    确认内置服务环境没有问题后 -- 产出代码到项目目录 或者 本地项目目录 使用：

    fis3 release -d prod ./output(./ouput  当前项目目录下的)

    prod （全局页面强制打包处理，打包结果在Pkg文件下）

3.
    每次prod 相当于版本更新一次 ，如果不手动清除之前的产出目录，则会造成上一个版本文件和最新

    版本文件共存 。这显然是不行的 ，暂时没有找到可以自动清除上一版本产出目录的方法 。所以暂时每次上新版本时

    手动清除一下 ！


4.   在js,CSS, html页面或者前端模板引擎中可以暂时不考虑map.json的生成 。不用每次在页面中注释html依赖的css或者js



5.  所有图片使用base64编码 。无论是在html还是css中后缀为?__inline 。
